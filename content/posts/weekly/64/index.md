### 2026 是自托管之年

> [!note] 📜
> [2026 is the Year of Self-hosting](https://fulghum.io/self-hosting)

作者表示，现在够买一个小巧的 Linux PC 已经很便宜，网络层面也有 Tailscale 提供支持，再加上 Claude Code 等基于命令行的 Agent 已经非常智能，那些缺少计算机系统和运维知识的人也能很方便地配置和使用自托管应用。你只需要买一个迷你主机放在家里，在这台主机和你的电脑上安装并配置 Tailscale，用你的电脑 SSH 到这台作为家庭服务器的主机上安装 Claude Code，然后告诉它你想要做什么就好了。

我自己也有一台利用率不高的 Mac Mini，还是 M4 芯片的，完全可以用来做家庭服务器。今年我也在逐步转向开源软件和自托管服务，我应该会参照作者的方案自托管密码管理器、照片管理和流媒体服务等。

不过，我有一点不解：使用 Anthropix 的 Claude Code 协助进行系统维护、使用一家加拿大公司提供的网络产品实现公网连接，听起来除了服务器是自己的（在购买了没有各种奇奇怪怪的锁和限制的设备的前提下），其他服务依然依赖科技公司，并没有很「自托管」。我最近在筹备迁移网站的一些基础设施，还打算自建 Fediverse 实例，当我重新审视我的网站架构时，我发现要脱离大科技公司真的很难，至少成本很高。比如，在我找到更好的防护方案前，我大概是不敢脱离 Cloudflare 的。话说回来，有了拥有数据、行使自由和拥抱开源技术的意识，就已经是很好的开始了。

### 全能型输入框

> [!note] 📜
> [A Box of Many Inputs](https://allenpike.com/2025/a-box-of-many-inputs/)

尽管我不使用并且抗拒使用任何现有的 AI 浏览器（在我看来开发 AI 浏览器是拿着锤子找钉子的做法，而且对隐私没有任何保障），不过这类浏览器中的地址栏设计很有意思。不少现代软件都内置了一个可以随时打开的输入框（常见的有 VS Code 里的 ⌘P 打开文件选择窗口，Obsidian 也可以用同样的快捷键打开命令选择窗口，Raycast 这类启动器软件更是提供了操作系统级别的全局输入框），这些输入框可以根据用户输入的内容推断意图，并提供合适的选项。AI 浏览器的地址栏也做了相同的处理，一些输入会被推断为问题，会直接打开 AI 对话框；另一些输入则被当作常规的搜索关键词。

作者用 Dia 浏览器搜索名为 *Who Framed Roger Rabbit?* 的电影，因为电影名本身是一个问句，结果被视作问题触发了 AI 对话——作者相信没有人在搜索电影的时候想要得到这样的结果。在 ChatGPT 的 Atlas 这边，这个问题并不会触发对话，而是会进入常规的搜索结果页面。区别在于，Dia 浏览器使用一个本地的小模型判断用户输入的内容到底是问题还是搜索关键词，而 Atlas 则简单粗暴地把 10 个单词以内的输入都当作常规搜索。（我很好奇这种判断法对中文的兼容性）

作者期待在未来，AI 浏览器会把输入框做得越来越好，能够更准确判断用户的意图。可我觉得这是设计的大忌。在 *The Design of Everyday Things* 这本书里，作者花了很长的篇幅批评以前的电话系统，座机上的一个按键能干的事情太多了，用户根本找不到某些功能的入口，不仔细拿说明书钻研，甚至会以为这个电话没有呼叫保持（Hold）功能。清晰比聪明更好，AI 对话和常规的搜索应该分开，至少设置明确的、不同的触发方式，比如回车是搜索，而 ⌘ 加上回车就是 AI 对话。用户使用软件的习惯不同，再精确的匹配和判断总有例外。不应该替用户做决定，而是要让用户自己做决定。

### 死互联网理论

> [!note] 📜
> [Dead Internet Theory](https://kudmitry.com/articles/dead-internet-theory/)

作者在 Hacker News 上冲浪的时候，发现了一个有趣的讨论串。有人质疑某个作者的开源项目的代码是 LLM 写的，因为他们找到了疑似 AI 生成的注释，更有趣的是，这个作者尽管反复强调他的代码是 100% 人工完成的，但他的回复中能看到很多 LLM 生成的痕迹，比如开头有那句臭名昭著的「You're absolutely right!」，末尾也有「Let me know if you...」这些人类绝对不会说的话，当然，还有经常被人诟病的对破折号的滥用。

作者想起了很早以前有人提出的「死互联网理论」（Dead Internet Theory[^1]），这个理论声称到了 2016 年，互联网上只会有相互对话的机器人，而所有的内容都不是真实的，只是人们为了卖东西用算法制造的内容。作者想起他在领英上看到一个雇员发照片宣传他们公司的工作氛围，大家都很开心，可很快就有人找到了 AI 生成的痕迹，因为背景上有毫无意义的文字、人物的手看起来怪怪的…… 看起来这个预言在晚了 10 年之后终于还是应验了。

尽管在以前，你也不知道在互联网另一边跟你聊天的人到底是人还是一只狗，但至少你知道他们是真实存在的。就算不讨论真实性，如今互联网的体验的确非常接近「人们只是为了卖东西给你而创造内容」的预言，真实且不商业化的内容已经越来越少了。就在这周，OpenAI 已经开始测试在 ChatGPT 内容中投放广告[^2]。我相信没有人喜欢广告，我讨厌广告的原因有很多： 一、广告是信息噪音，内容中参杂了不必要且夸大的信息；二、广告生来就不是为了陈述事实的；三、为了更精准地投放广告，服务商会收集并分析用户数据，这是对隐私的极大侵犯。

这也是我今年更彻底地与「大公司网络」决裂，拥抱 IndieWeb 和 Fediverse 的原因。我作为用户，我理应有各方面的决定权，去中心化、自托管和开源能让我更好地行使这些权利。互联网已经混乱不堪了，让我们回到更小但更开放且更自由的网络上来。

### 预测：微软将会在未来停用 Windows

> [!note] 📜
> [Prediction: Microsoft Is Going To Do The Funniest Thing Imaginable](https://gamesbymason.com/blog/2026/microsoft/)

听起来有些荒诞不经的想法：作者预计在接下来 15~25 年内，微软会停止维护 Windows，开发 Windows 主题的 Linux 发行版。

以下是作者的理由：

1. 所有人都知道 Windows 越来越差，比如前不久 Windows 还让国外出现了大规模的蓝屏和系统瘫痪。作者以前觉得桌面端系统有三个可行的选择，也就是 Linux、Windows 和 macOS，作者作为程序员，现在已经不认为 Windows 能够作为一个选项了。作者表示习惯了用 Windows 系统的都该去试试别的操作系统，而不是习惯了电脑的各种问题，认为 PC 就应该是这样的。
2. 与此同时，Linux 一直在进步。人们对 Linux 的刻板印象是界面和操作不够友好，但这现在已经不是问题了。我最近很有感触的一点是，我甚至没有办法在 macOS 上找到一个像 Linux 上的 Hyprland 一样好用的平铺窗口管理器，大概是因为 macOS 把相关权限卡得太死。
3. 游戏体验在 Linux 上已经更好了。Valve 的 Steam Deck 就是 Linux 系统，而且上面能使用基于 Wine 的 Proton 兼容层运行 Windows 游戏，并且性能不差。在写这篇周刊的时候，Wine 也发布了 11.0 版本，Windows 上的老代码运行在 Linux 上并无大碍。微软如果真的要开始使用 Linux 发行版，也不会面临**太严峻**的重建生态的问题。
4. 所有人都会听游戏玩家的话。大部分人对电脑的需求不高，如果他们预算充足，想要一台足够快的电脑，多半会选择 MacBook 或者买游戏本。一旦游戏玩家们迁移到 Linux，市面上有越来越多搭载 Linux 系统的游戏 PC，也会有越来越多的人开始使用 Linux 系统，而且，如果界面没有太大变化，大多数人可能都不会注意到操作系统变了。

尽管我很早就脱离了 Windows，也几乎不用微软产品（甚至觉得微软的 Office 套件是毒瘤），但要是能看到 Linux 系统统治世界，我还是很期待的。那就…… 再等十几年看看？

### AeroSpace

macOS 上的一个平铺窗口管理器。所谓平铺式窗口，就是「不让窗口互相重叠」。这类软件极大地限制了桌面的灵活度，但带来了高效的操作和绝对的整洁。AeroSpace 使用 Vim 键位操作窗口，按下 `⌥h` 可以将焦点移动到左侧的窗口，`j` `k` `l` 也同理；在这个操作之上加上 `Shift` 键，就可以将窗口移动到指定的方位。把 `h` `j` `k` `l` 换成数字或其他字母，可以对「工作区」（workspace）进行管理，切换到某个工作区或将聚焦的窗口移动到某个工作区。

使用 AeroSpace 可以很方便地排列窗口，更有用的是，形成自己的习惯之后能够快速地在不同使用场景之前切换。比如我的 1 号工作区是运行 NeoVim 的终端和另一个用来干杂货的终端，我的 S 号（Social）工作区打开了 Mastodon 客户端、Matrix 客户端和 NetNewsWire（RSS 阅读器），D 号（Dashboard）工作区是打开了 Cloudflare 仪表盘、Spaceship 管理界面和各种后台的网站。

如果你也是 macOS 用户，我非常推荐这个软件，能够极大地提高效率。

访问：[AeroSpace](https://github.com/nikitabobko/AeroSpace)

### 合租四个月后的感受

我宜人性不佳，会讨厌和自己生活习惯相冲的人。能够协商，妥协后仍然会讨厌，尽管大部分时候不会让情绪影响正常的生活关系。协商对我来说是消耗精力的事情，各退半步的结果是没有人满意，却要装作满意，这种不得已的虚伪令我不适。

去年十一月和十二月我都安排了公共区域的清洁，同室（原谅我不使用「室友」这个词）夸奖了我的安排清晰，尽管他明显对「每天都要用吸尘器清洁主要生活区域」这项安排感到不解，却没有提供实在的意见。他的态度其实很明显了，打卡未完成的情况不少，而且他对环境的要求远不如我：衣物常被发现于客厅沙发；灶台和抽油烟机这种大家伙不便分配，也是我完成的；我还在雾霾天的早晨发现被他打开的窗户，家不远处就是建筑工地，仅仅是打开书房门一小会儿挺能听到我的空气净化器轰隆作响。

我知道主要原因是我对环境的要求太高了（尽管我认为这是基本的个人卫生问题，但遗憾的是，似乎在大部分人眼里我十分吹毛求疵），我平时待人友善，但面对生活环境免不了在心里暗暗骂人。

这个月我不打算继续做安排了，拿自己的标准要求别人总归不合适，非要保全公平性反而让我和对方都不舒服，于是默默回到「谁先看不下去谁先干活」的模式，各自都优先收拾自己看重的部分，反倒少些不爽。

时常想回到独居生活，可惜没钱，只能作罢。
